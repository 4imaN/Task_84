# LedgerRead API Specification

## 1. Authentication And Workspace Entry

### REST

- `POST /auth/login`
  - Validates username, password, and target workspace.
  - Resolves the account through the encrypted-at-rest username lookup hash instead of querying plaintext usernames.
  - Sets an `httpOnly` session cookie and returns user role plus workspace home route metadata.
- `POST /auth/logout`
  - Invalidates the active session and clears the session cookie.
- `GET /auth/session`
  - Validates the current cookie/session context and returns session + role metadata.
  - Includes a request `traceId` for local troubleshooting correlation.

## 2. Customer Catalog And Reader Fetch

### GraphQL

- `catalog`
  - Returns featured titles, best-sellers, and recommendation snippets.
- `title(id)`
  - Returns title metadata, TOC, author/series details, reader preferences, and both Simplified/Traditional chapter bodies for instant client-side language switching.
- `communityThread(titleId)`
  - Returns comments, replies, ratings summary, viewer-specific masking, and hydrated favorite/author-follow/series-follow state for the active reader.

## 3. Reading Profile Portability

### REST

- `GET /profiles/me`
  - Returns the persisted reading profile for the authenticated user.
- `PUT /profiles/me`
  - Updates preferences and sync timestamp.
  - Rejects stale incoming timestamps with `409` when a newer server profile already exists.
- `POST /profiles/me/sync`
  - Accepts a LAN-sync payload and resolves or rejects stale timestamps with `409`.
  - The profile surface is intentionally self-only; there is no cross-user read endpoint.

### Client-Side Only

- Reading-profile export/import file handling
  - Runs entirely in the browser with local encryption/decryption.
  - Imported files are validated for JSON shape and password correctness before conflict-aware sync runs.
  - Older imported files are ignored locally, and newer files are sent through `POST /profiles/me/sync` so the server can apply newest-timestamp conflict resolution.
  - When profile fetch fails and no cached profile is available, the UI renders a retryable error state instead of remaining in a non-terminal loading state.

## 4. Community Actions

### REST

- `POST /community/comments`
  - Creates comments or replies with anti-spam and sensitive-word checks.
  - If `parentCommentId` is present, the parent comment must belong to the same `titleId`; cross-title replies are rejected.
- `POST /community/reports`
  - Reports content with required category and notes.
  - `category` and `notes` must be non-empty after trimming; whitespace-only payloads are rejected with `400`.
- `POST /community/relationships/block`
  - Blocks or unblocks another user.
- `POST /community/relationships/mute`
  - Mutes or unmutes another user.
- `POST /community/ratings`
  - Creates or updates a 1-5 rating.
- `POST /community/favorites`
  - Adds or removes a title from the reader's favorites.
- `POST /community/subscriptions/authors`
  - Subscribes or unsubscribes the current reader to an author.
- `POST /community/subscriptions/series`
  - Subscribes or unsubscribes the current reader to a series.

## 5. POS Cart Construction And Review

### REST

- `GET /pos/search`
  - Searches inventory by SKU/name for clerk cart building.
  - Supports optional `?q=<term>` filtering and returns inline stock plus bundle-suggestion context used by the POS workflow.
- `POST /pos/carts`
  - Starts a cart.
- `POST /pos/carts/:cartId/items`
  - Adds SKU quantities, clears any prior review marker, and returns bundle suggestions plus inline stock feedback.
- `PATCH /pos/carts/:cartId/items/:cartItemId`
  - Sets an exact line-item quantity, clears the prior review marker, and returns the updated cart summary.
- `DELETE /pos/carts/:cartId/items/:cartItemId`
  - Removes a cart line, clears the prior review marker, and returns the updated cart summary.
- `POST /pos/carts/:cartId/review-total`
  - Revalidates inventory, pricing, discounts, and fees before checkout.
  - Persists a signed review snapshot on the cart.
- `POST /pos/carts/:cartId/checkout`
  - Records offline payment method and finalizes inventory mutation.
  - Rejects the request unless a fresh successful review exists and the live cart still matches the reviewed signature.

## 6. Attendance And Governance

### REST

- `POST /attendance/clock-in`
  - Creates a clock-in event with optional evidence file.
  - Evidence files are validated by binary signature, not MIME alone, and stored under server-generated filenames inside the evidence root.
- `POST /attendance/clock-out`
  - Creates a clock-out event and re-evaluates risk rules.
- `GET /attendance/risks`
  - Returns active alerts for authorized internal roles.
  - Missing clock-out alerts are also generated by a scheduled backend evaluator every 30 minutes, so overdue risks appear even if the affected user never polls this endpoint.
- `GET /moderation/queue`
  - Returns moderator queue records.
  - Supports `?status=OPEN|RESOLVED|ALL` so moderators can work active reports and resolved restore history separately.
- `POST /moderation/actions`
  - Applies hide, restore, remove, or suspend actions and emits immutable logs.
  - Suspend derives the target account from the reported comment/report context on the server, so moderators cannot silently resolve a report without suspending the actual reported user.

## 7. Reconciliation And Audit Review

### REST

- `POST /admin/manifests/import`
  - Authorized roles: `MANAGER`, `INVENTORY_MANAGER`
  - Imports supplier statements and matched supplier invoices in one transaction.
  - Persists statement lines, invoice lines, landed-cost allocations, inventory receipts, and payment-plan status.
  - Recalculates moving-average cost from purchase cost plus allocated freight/surcharges.
  - Raises discrepancy flags when statement vs invoice quantity differs by `2+` units or amount differs by more than `$5.00`.
- `GET /admin/settlements`
  - Returns settlement status, payment plan states, statement/invoice references, landed-cost totals, and discrepancy summaries.
  - Supports `?status=PENDING|MATCHED|...` filtering.
  - Powers both `/admin/finance` and `/finance/settlements`, which share the same reconciliation review UI for settlement status, discrepancy flags, and audit-trail access.
- `GET /admin/audit-logs`
  - Returns hash-linked audit events chronologically.
  - Supports `?limit=<n>&action=<AUDIT_ACTION>` pagination and action filtering.

### Frontend Route Policy

- `/admin/overview`
  - `MANAGER`, `INVENTORY_MANAGER`
- `/admin/finance`
  - `MANAGER`, `INVENTORY_MANAGER`
- `/admin/inventory`
  - `MANAGER`, `INVENTORY_MANAGER`
- `/admin/audits`
  - `MANAGER`, `INVENTORY_MANAGER`
- `/finance/settlements`
  - `FINANCE`
  - Reuses the reconciliation review surface that managers and inventory staff see at `/admin/finance`.
  - Settlement/discrepancy review stays available here, but manifest import controls are intentionally absent because the mutation is restricted to manager and inventory roles.
- `/finance/audits`
  - `FINANCE`

## 8. Recommendations

### GraphQL

- `recommendations(titleId)`
  - Returns Top-N and similar items from the nightly snapshot cache.
  - Every response path writes a PostgreSQL trace row, including cache hits and best-seller fallbacks.
